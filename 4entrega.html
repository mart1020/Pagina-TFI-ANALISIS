Prototipos

Prototipos NO operacionales

Como parte del Proceso Unificado, se desarrollaron prototipos no operacionales (mockups) con el objetivo de validar los requerimientos funcionales antes del diseño detallado e implementación. Estos prototipos permitieron a los actores internos revisar:

  • El flujo de interacción con el sistema.
  • Los datos requeridos en cada operación.
  • El comportamiento esperado ante validaciones y restricciones.
  • La consistencia entre los Casos de Uso y la interfaz.
  • La detección temprana de errores o requisitos faltantes.

Prototipos desarrollados:

Pantalla de Alta de Empleado
  • Etiquetas descriptivas para cada dato solicitado.
  • Cuadros de texto para apellido, nombre, DNI, dirección, etc.
  • Controles para validación de formato (DNI, fechas).
  • Lista desplegable para seleccionar el estado inicial.
  • Botón de comando Guardar.
Pantalla de Modificación de Empleado

Permite:

  • Ingresar un legajo existente para su búsqueda.
  • Visualizar los datos actuales del empleado.
  • Editar únicamente los campos habilitados para modificación.
  • Validación gráfica de datos inconsistentes.
  • Botones de comando Guardar y Cancelar.
Pantalla de Consulta de Empleados

Incluye:

  • Filtros por campo: legajo, apellido, área y estado.
  • Listas desplegables para selección de criterios.
  • Botón de comando Buscar.
  • Tabla de resultados con columnas Legajo, Nombre, Apellido y Estado.
  • Mensajes informativos para situaciones de “sin resultados”.
Pantalla de Carga de Documento

Incluye:

  • Campo para ingresar el número de legajo.
  • Lista desplegable con los tipos de documentos permitidos.
  • Selector de archivos.
  • Validación visual de extensión y tamaño permitido.
  • Mensajes de error ante documentos no válidos (formato incorrecto, tamaño excedido).
  • Vista del historial de documentos cargados.
  • Botón de comando Subir.
Pantalla de Cambio de Estado

Incluye:

  • Ingreso de legajo para búsqueda.
  • Visualización del estado actual del empleado.
  • Lista desplegable con los estados permitidos.
  • Cuadro de texto para ingresar el motivo del cambio.
  • Botón de comando Confirmar.
  • Mensajes de advertencia en caso de transiciones no permitidas
